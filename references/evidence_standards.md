# 证据标准 — 双轴溯源体系 v1.7

> **v1.7.1**：新增 `tags` 检索字段——FTS5 无 CJK 分词器，中文多词 AND 搜索需英文/拼音桥接。
> **v1.7.0**：置信度标签拆分为双轴——**验证状态**（事实性）+ **优先级**（行动性）。

## 数据字段

| 字段 | 问什么 | 值 |
|------|------|------|
| `verified` | 证实了没？ | `confirmed` / `pending` / `excluded` |
| `priority` | 多紧急？ | `high` / `medium` / `low` |
| `source` | 谁说的？ | KP旁白 / 角色名 / 骰子 / 场外(玩家名) |
| `tags` | 怎么搜？ | 英文/拼音+中文关键词数组（FTS5 检索桥） |

**互不覆盖**——四个字段各自独立。`tags` 是技术字段，不影响证据法律效力。

## 验证状态

| key | 显示 | 含义 |
|------|------|------|
| `confirmed` | 🟢 已证实 | 多方证据支撑，可作事实引用 |
| `pending` | 🟡 待验证 | 单方陈述或初步发现 |
| `excluded` | 🔴 已排除 | 经调查后证伪，不入 SQL |

**状态升级路径**：`pending → confirmed`（积累证据）或 `pending → excluded`（证伪）。

## 优先级

| key | 显示 | 含义 |
|------|------|------|
| `high` | 🔴 优先 | 不跟这条线索会丢关键信息 |
| `medium` | 🟡 普通 | 常规调查项 |
| `low` | ⚫ 可延后 | 背景信息/非主线 |

**优先级与验证状态无关**——已排除的线索可能是高优先级（因为排除了一个关键猜想）。

## 来源标签（不变）

| 来源 | 含义 |
|------|------|
| `KP旁白` / `KP场景` | 最高可靠——KP 直接给出的世界事实 |
| `角色名` | 该 NPC 的陈述（可能主观/片面） |
| `骰子` | 技能检定结果（含成功/失败） |
| `场外(玩家名)` | 玩家推理/直觉，⚠️ OOC需标注 |

## 来源跨越规则（不变）

| 从 | 到 | 条件 |
|----|----|------|
| 推测 → 场内 | ✅ | 需要新证据支撑 |
| 推测 → 铁证 | ❌ | 不能直接跳级——必须先成为场内确认信息 |
| 场外 → 场内 | ✅ | 游戏内角色获取到了对应信息 |
| 场内 → 铁证 | ✅ | 积累足够证据 + KP确认 |

## 防超游防火墙（不变）

| 层 | 可代入角色？ |
|----|:--:|
| verified=confirmed + source=场内 | ✅ |
| verified=pending + source=场内 | ✅（标注验证状态） |
| source=场外 | ❌ 禁止带入角色知识 |
| verified=excluded | ❌（已证伪） |
